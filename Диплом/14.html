Виртуальные частные сети

Основные понятия. Виртуальная частная сеть (Virtual Private Network — VPN) является подсетью корпоративной сети, обеспечивающей безопасное вхождение в нее удаленных пользователей. Виртуальная сеть — это сеть, которая накладывается на реальную физическую сеть путем создания виртуальных каналов «источник—получатель». Виртуальный напал функционально эквивалентен выделенному логическому соединению «точка—точка». Маршрут передачи данных не фиксирован, поэтому виртуальный канал может иметь различную конфигурацию в реальной сети. Данные передаются от узла к узлу, последовательно формируя двухточечные соединения по способу инкапсуляции данных. Суть инкапсуляции заключается в следующем: в пакет, передаваемый через транзитные узлы сети, вводятся дополнительные заголовки, которые после прохождения каждого узла удаляются. Таким образом, технология VPN создает логическую сеть, независимую от места расположения терминальных устройств и непосредственных физических соединений. Виртуальный канал обычно создается в публичной сети (Internet) и называется туннелем [9]. Частная сеть требует принятия дополнительных мер по защите передаваемой по VPN информации. Требование конфиденциальности особенно важно, потому что пакеты, передаваемые по публичной сети, уязвимы для перехвата при их прохождении через каждый из узлов (серверов) на пути от источника к получателю. Его реализация возлагается на туннели, которые должны поддерживать проверку подлинности(аутентификацию) пользователей и шифрование данных при их передаче с одного конца туннеля на другой. Туннелирование объединяет три процесса: инкапсуляцию, маршрутизацию по туннелю и извлечение исходного пакета данных. Инкапсуляция скрывает исходный пакет данных внутри другого пакета. При этом конечный адрес получателя (принимающего компьютера) расположен внутри (в заголовке) исходного пакета. выбора маршрута по туннелю в заголовке каждого нового (внешнего) пакета, заголовком инкапсуляции, размещается адрес конца туннель. Когда пакет достигает конца туннеля, заголовок инкапсуляции удаляется. После нескольких ретрансляций исходный пакет поступает на принимающий компьютер. Виртуальные частные сети VPN создаются для достижения одной из следующих целей: • предоставления удаленного доступа индивидуальным пользователям (например служащим, находящимся в командировке), реализация которого может быть условлена такими факторами, как разные операционные системы и протоколы, установленные на клиентских компьютерах, при этом клиентские компьютеры VPN должны поддерживать протоколы, используемые сервером VPN (протоколы туннелирования, сетевой и транспортный) и методы шифрования, принятые в данной VPN; • создания экстрасети (это такая часть локальной сети компании, в которой могут храниться технические материалы для заказчиков или совместно используемые документы для партнеров компании, т. е. данные, не требующие защиты: остальная часть внутренней сети должна быть защищена от внешнего доступа) ; • коммуникации двух офисов, расположенных на большом расстоянии, без затрат на выделенные прямые каналы. Способы реализации VPN. Сеть VPN может быть реализована программным и аппаратным способами. При программной реализации используются протоколы туннелирования и различают два типа сетей. 1. Сети на основе программного обеспечения независимых поставщиков. Этот вид программного обеспечения обычно включает дополнительные средства, расширяющие диапазон использования сетей VPN. Многие программы поставщиков предоставляют широкий выбор средств безопасности, которые можно легко реализовать. Некоторые программы позволяют создавать туннели, которые на основе оборудования создать затруднительно. 2. Сети на основе программных средств, встроенных в операционную систему. Средства создания VPN, включенные в современные операционные системы, такие как Windows 2000, оказываются достаточными для решения большинства задач. Их очевидным преимуществом является меньшая стоимость, поскольку не нужно покупать дополнительного программного обеспечения. При аппаратной реализации различают два вида сетей VPN. 1. Сети на основе маршрутизаторов. Основой этих VPN являются маршрутизаторы с встроенными средствами шифрования. Они обладают самой высокой сетевой производительностью, к тому же их обычно легко устанавливать и использовать. 2. Сети на основе брандмауэров, обладающие повышенной безопасностью. Их недостатком является пониженная производительность. Для ее повышения используют специализированные процессора, ускоряющие шифрование данных. Брандмауэр является средством защиты, имеет усиленную аутентификацию и детализированную регистрацию, обеспечивает фильтрацию входящих и исходящих пакетов. На основе правил безопасности, установленных сетевым администратором, брандмауэр определяет, следует лм пропустить поступивший пакет. Поддержка сетей VPN. Современные операционные системы содержат встроенные средства для программной поддержки виртуальных частных сетей, например, Windows 95/98 и Windows NT/2000, которые могут работать как клиенты VPN. Так, с помощью своих встроенных компонентов Windows 2000 поддерживает протокол туннелирования «точка-точка» (Point-to-Point Tunnelinq Protocol — PPTP), обеспечивающий создание криптографически защищенных соединений путем инкапсуляции пакетов, управление потоками данных и многопротокольное туннелирование. В состав Windows 2000 включен также мастер, который помогает пользователю пройти по этапам установки соединения VPN. Кроме того, широко применяется сетевая операционная система Linux, которая поддерживает протоколы PPTP и IPSec (IP Security — Internet-протокол конфиденциальности), обеспечивающий аутентификацию, проверку целостности и шифрование пакетов. При протокольной поддержке V Р N используются три типа протоколов. 1. Протоколы для создания туннеля (например, протокол PPTP), которые инкапсулируют данные таким образом, что заголовки исходных протоколов оказываются внутри инкапсулирующих заголовков. 2. Протоколы шифрования, или безопасности, позволяющие зашифровать данные для обеспечения их безопасности, когда туннель создан. В сетях VPN компании Microsoft совместно с PPTP используется протокол шифрования MPPE — Microsoft Point to Point Eпcryption. 3. Сетевые/транспортные протоколы, которые иногда называются локальными протоколами. Безопасность VPN. Безопасность сетей VPN обеспечивают следующие процедуры: аутентификация, авторизация и шифрование. Аутентификация используется для проверки подлинности пользователя сети VPN и может выполняться на уровне компьютеров, например, путем обмена сертификатами компьютеров. Авторизация предназначена дал ограничения числа пользователeй, которым предоставляется доступ в ceть VPN. Доступ предоставляется в соответствии с принятой стратегией безопасности, которая определяет, кто из пользователей может получать доступ в VPN, а кому в доступе должно быть отказано. Шифрование обеспечивает безопасность данных путем их кодирования. Незашифрованные данные уязвимы для перехвата злоумышленниками в процессе их передачи по публичной сети. Многие технологии VPN позволяют выбирать метод шифрования. Такая многоуровневая система мер безопасности обеспечивает высокую конфиденциальность данных, передаваемых по VPN.
