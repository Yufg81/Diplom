Гибридные сети

Сеть, в которой используются разные операционные системы, типы оборудования, протоколы и службы, называется гибридной сетью. В современном корпоративном мире даже небольшие локальные сети являются гибридными по всем перечисленным признакам, не говоря о такой глобальной сети, как Internet. Причины появления гибридных сетей. Несмотря на то, что сети, построенные на однотипном оборудовании, на основе одной операционной системы и с одним стеком протоколов, обладают определенными достоинствами, существуют многочисленные причины, побуждающие включать в сеть компоненты разных поставщиков. К таким причинам относятся невозможность использования при построении сети компонентов одного поставщика из-за отсутствия в его ассортименте всех компонентов с требуемыми характеристиками, или личные предпочтения пользователей (пользователи управленческого уровня предпочитают установку в сеть Windows 2000 клиентского компьютера Linux), а также финансовые соображения, например, запланированный переход на новую операционную систему не может быть осуществлен из-за высокой стоимости или нехватки финансовых средств. изучения гибридных сетей целесообразно выделить один признак и последовательно рассмотреть три разновидности сетей: 1) сети с разными протоколами; 2) сети с разными операционными системами (как серверными, так и клиентскими); 3) сети на основе нескольких платформ с разнотипным оборудованием. Сети на основе многих протоколов. Многоуровневая сетевая модель требует для нормального функционирования сети большого числа протоколов. При этом обычно имеют в виду использование в сети более одного протокола сетевого/транспортного уровня или более одного протокола разделения файлов. Рассмотрим особенности каждого из этих случаев. Протоколы сетевого/транспортного уровня. Можно назвать несколько причин, заставляющих использовать в сети более одного стека протоколов сетевого/транспортного уровня. Одной из таких причин является наличие двух и более сетевых операционных систем. Чаще всего необходимость в нескольких протоколах возникает в гибридных сетях Microsoft/NetWare. В сетях с операционной системой NetWare 5. х клиенты связываются с серверами Microsoft с помощью TCP/IP. Однако имеющееся в корпоративных сетях большое чисто серверов с операционными системами NetWare 3. х и 4. х требует использования стека протоколов IPX/SPX. Поэтому у клиентов NetWare 5. х для доступа к файлам серверов NetWare 3. х или 4. х до но быть установлено не только клиентское программное обеспечение Novell или Microsoft, но и реализации стека протоколов IPX/SPX. Другая причина заключается в необходимости установки соединений с Internet. Для установки локальных соединений в небольшой сети Microsoft целесообразно использовать протокол NetBEUI, достоинством которого является быстродействие, простота конфигурирования и устранения неполадок. Однако для соединения компьютеров такой локальной сети с Internet должен использоваться стек протоколов глобальных сетей TCP/IP. Также следует учитывать соображения безопасности. Чтобы повысить безопасность сети при подключении ее к Internet, некоторые сетевые администраторы устанавливают во внутренней части сети стек сетевых/транспортных протоколов, отличный от TCP/IP, например, IPX/SPX. В этом случае компьютеры за пределами внутренней сети, не использующие IPX/SPX, не смогут получить доступ к компьютерам IPX/SPX, поскольку у них отсутствует общий протокол. Если с одним сетевым адаптером связано несколько стеков протоколов сетевого/транспортного уровня, то большое значение имеет порядок их связывания. Спецификации интерфейса сетевых драйверов (Network Driver Interface Specification — NDIS) компании Microsoft и открытый интерфейс связи (Open Datalink Interface — ODI) компании Novell допускают связывание многих протоколов с одним сетевым адаптером, и наоборот — многих сетевых адаптеров с одним протоколом. Протоколы разделения файлов. Для совместного использования файлов существует несколько видов протоколов: • в сетях Microsoh — протокол SMB (Server Message Block — блок сообщений сервера), который отвечает за структуризацию запросов и связь с различными операционными системами: • сетях NetWare — протокол NCP (NetWare Core Protocol — протокол ядра NetWare), который используется сервером предоставления сетевых услуг печати клиентам NetWare: • системе UNIX — протокол NFS (Network File System — сетевая файловая система), который позволяет совместно использовать файлы в локальной сети, избавляя от необходимости держать многочисленные копии файлов на локальных дисках отдельных компьютеров. Сети с несколькими операционными системами. Такие сети разделяют на три типа. 1. Сети с несколькими серверными операционными системами. Сеть может содержать несколько серверов с установленными на них разными операционными системами. Создание среды для доступа пользователей к фaйлам и другим нужным им ресурсам независимо от операционной системы возлагается на администратора сети. Для обеспечения доступа к разным серверам используются соответствующие пакеты программ. 2. Сети, в которых исполъзуется одна серверная и несколъко клиентских операционных систем. 3. Сети, в которых исполъзуется несколъко серверных и несколъко клиентских операционных систем. Сети на основе нескольких платформ. Под термином платформа будем понимать тип сетевой архитектуры или оборудования. При этом следует иметь в виду, что на разных платформах часто (хотя и не всегда) выполняются разные операционные системы. Например, платформа персоналъных IBM-компьютеров предполагает архитектуру и оборудование, отличающиеся от платформы Macintosh или платформы мэйнфреймов IBM. Необходимо отметить следующие особенности сетевой коммуникации: сетевая коммуникация между персональными IBM-компьютерами и компьютерами Macintosh реализуется относительно просто, посколъку они близки по производителъности, и сводится к взаимодействию между операционными системами, а для взаимодействия между персоналъными IBM-компьютерами и IBM-мэйнфреймами чаще всего используется архитектура сетевых систем (Systems Network Architecture — SNA), которая специалъно разработана корпорацией IBM для этих целей. В качестве способа гибридного соединения может быть предложен шлюз, представляющий собой аппаратно-программное средство (устройство) для соединения разнотипных сетей, работающих с различными протоколами связи. При передаче данных шлюз преобразует их в формат, совместимый с протоколом сети назначения. Программное обеспечение шлюзов позволяет связывать две сети разного типа, т. е. служит связующим звеном между компьютерами, в которых исполъзуются разные протоколы, операционные системы и платформы.
